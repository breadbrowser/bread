LLM Parameter Counting

2022-03-30 (3 min read)

Each weight or parameter is a float that was tuned during training and is usually two bytes as most training is done half-precision now(bfloat16). Not everything is trained/served bfloat16, but it's at least half-precision (at least since the GPT-3 Paper in 2020) which gets us the two bytes.

The weights loosely consist of the following, per each block (where one block a decoder unit that consists of a self-attention layer and a feedforward layer, though I'll refer to blocks as layers):

  • Wq,Wk,Wv W_q,W_k,W_v Wq​,Wk​,Wv​ matrices, which are each dmodel⋅nheads⋅dheadd_\text{model} \cdot n_\text{heads}\cdot d_\text{head} dmodel​⋅nheads​⋅dhead​ and project the input into the query, key, and value used in self-attention.
  • A Wo W_o Wo​ matrix, which is also dmodel⋅nheads⋅dheadd_\text{model}\cdot n_\text{heads}\cdot d_\text{head} dmodel​⋅nheads​⋅dhead​ and used on the output of self-attention, before the MLP layer (the feed-foward neural network that's stacked on the self-attention layer).
  • MLP weights, which are two matrices each of dmodel2⋅4{d_\text{model}}^2 \cdot 4dmodel​2⋅4. You might also see this referred to by feedforward or linear layers.

The four in the MLP weights calculation is based on architecture, but basically every transformer since the original from 2017 has gone with that ratio — where the MLP is 4 four times the size of the model embedding dimension. In a vast majority of transformer architectures, nheads⋅dhead=dmodeln_\text{heads}\cdot d_\text{head} = d_\text{model}nheads​⋅dhead​=dmodel​. You can see this in all the GPT models at Table 2.1 (the 13B model is off by 20, but might just be a... typo?), in the Gopher models in Table 1 (where what I called dheadd_\text{head}dhead​, they called "Key/Value Size"). This is not necessarily the case, but can be assumed.

This is not quite 52B. It's probably cheating to round up by half a billion parameters, but we can account for them! The equation above is most of the parameters, but we're missing token embeddings. Anthropic uses a 65536 vocab size, so we get ntokens∗dmodel=536,870,912n_\text{tokens} * d_\text{model} = 536,870,912 ntokens​∗dmodel​=536,870,912. We actually have that half a billion params twice for the unembeddings, which leads us to about 52.5B tokens.

We're also missing biases that are attached to all the weights, as well as layernorm. Biases should be approximately zero, and layernorm are dmodeld_\text{model}dmodel​ (though they exist per block), but otherwise known as zero. Transformers also have positional encoding mechanisms, which for GPT-2 and the original transformer is nctx⋅dmodeln_\text{ctx}\cdot d_\text{model}nctx​⋅dmodel​ (aka, zero) but Gopher 280B there's 21.5B weights spent on the relative positional encoding method presented in the Transformer XL paper.
